# Module 00 — Getting Started

## What You Will Learn

- What Linux actually is (in plain English).
- Why Linux is worth learning, especially for DevOps/Cloud/SRE.
- Where Linux is used in the real world.
- A clear roadmap for the rest of this repository.

## Why This Module Matters

Before typing a single command, you need the **big picture**. Understanding *why* Linux dominates servers and the cloud gives every later command meaning. Skipping the "why" is the #1 reason beginners give up.

## Real-World Use Case

When you log into an AWS EC2 instance, a Docker container, or a Kubernetes node, you are using Linux. This module explains why those systems chose Linux — so the later modules feel relevant, not abstract.

## Hands-On Practice

There is little to type in this module — it's conceptual. Your one task: **decide your practice environment** (WSL, VirtualBox, or cloud) so Module 01 is ready to go.

## Common Mistakes

- Rushing past the concepts to "just learn commands." The concepts make the commands stick.
- Believing Linux is "too hard." It's just unfamiliar — that fades fast with practice.

## Troubleshooting

Nothing to break yet. If terms feel overwhelming, re-read the **Simple Layman Explanation** sections — they're written for total beginners.

## Best Practices

- Read each file fully once, then skim it again before moving on.
- Keep a personal notes file of new terms.

## Quick Revision

- Linux is a free, open-source operating system that powers most servers and the cloud.
- It's worth learning because nearly all modern infrastructure runs on it.
- This repo takes you from zero to intermediate, module by module.
